GraphQL

GraphQL

  • Graph Query Language
  • facebook에서 만든 쿼리 언어로, API 구현에 사용한다.
  • 기존에 API를 만드는 방법으로 RESTful 를 사용하곤 하지만 이 방식에서 불편함을 느껴 만들었다고 한다. (참고 : https://graphql.org/blog/graphql-a-query-language/)
    • 다양한 기종(웹, 모바일 등)에서 일일히 기능을 구현하는 것이 비효율적

REST API 와의 비교

REST API와 GraphQL 방식은 각자 장/단점이 있을 것이고 서비스에 맞게 선택하면 될 것 같았다.

응답

  • RESTful API
    • response 구조가 정해져있어 사전 규약대로만 응답을 받을 수 있다.
  • GraphQL
    • 요청 시, 사용자가 원하는 응답의 구조로 얼마든지 변경 가능하다.

End-point

  • Restful API
    • Resource 마다 하나의 End-point를 가진다.
  • GraphQL
    • 하나의 End-point 를 가진다.
