2019-04-22
Java

Collection

Collection Framework

  • 다수의 데이터를 쉽고 효과적으로 처리할 수 있는 표준화된 방법을 제공하는 클래스의 집합

Collection Class

  • 데이터를 저장하는 자료 구조와 데이터를 처리하는 알고리즘을 구조화하여 클래스로 구현해 놓은 것
  • 인터페이스(interface)로 이미 구현이 되어있어서 적재적소에 사용하면 됨 🙇
  • 제네릭(generic)으로 표현

Generic

데이터 타입을 일반화한다.

  • 클래스/메소드 내부에서 사용되는 객체의 안정성 보장
  • 이미 타입이 명시되어 있기 때문에 반환 값에 대한 타입 검사가 필요 없다!

주요 인터페이스

List

  • 요소의 저장 순서가 유지되며 중복을 허용한다.
  • ArrayList, LinkedList(single/double), Stack, Queue, Vector(사용도 낮음)
  • 예제 코드 ArrayListSample.java

    ArrayList

  • Array vs ArrayList
    • Array : 초기화 시 크기를 지정한다.
    • ArrayList : 크기가 가변적이다.
  • 인덱스를 사용하여 데이터를 빠르게 찾을 수 있다.

Map

  • key/value 로 이루어진 데이터의 집합
  • HashMap, TreeMap, Hashtable, Properties
  • 예제 코드 MapSample.java

HashMap / Hashtable

  • Map 컬렉션 - HashMap, Hashtable
  • HashTable 개념 5분만에 이해하기

Set

  • 순서가 없으며 중복 허용하지 않음
  • HashSet, TreeSet

정리하면서 느낀 점

  • 컬렉션이 탄생한 배경에 대해 생각해보게 되었다!
    => (Based) 많은 양의 데이터를 어떻게 효율적으로 다룰지에 대한 고민
  • 컬렉션을 사용하기 전, 무엇을, 어떻게?를 생각하며 설계할 것
