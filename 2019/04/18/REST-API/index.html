REST API

2019-04-18
Tech

들어가기 전

기존에 운영 중이던 API 시스템을 개선하고자 검토했던 내용이다.
당시 REST의 개념을 정확히 알고 있지 못한 상태여서 개념부터 차근차근 정리했었는데 기록으로도 남겨보자 싶었다.

REST(Representational Safe Transfer)

  • HTTP URL로 표현된 리소스에 대한 행위를 HTTP Method로 정의한다.

    리소스, 메소드, 메시지로 구성

  • 여기서 리소스(Resource)는 시스템 or 서비스마다 정의하는 방법이 다양해질 수 있다.

    개인적으로 이거 정의하기가 너무 어려웠다! 귀에 걸면 귀걸이, 코에 걸면 코걸이..

  • Stateless : 클라이언트의 데이터를 서버에 의존하지 않도록 구현 > 단순한 기능을 수행하는 API 작성

구성 요소

  • 자원(RESOURCE) - URI
  • 행위(Verb) - HTTP METHOD
  • 표현(Representations)

특징

  • URI는 정보의 자원을 표현해야 한다.
  • 자원에 대한 행위는 HTTP Method(GET, POST, PUT, DELETE)로 표현한다.
