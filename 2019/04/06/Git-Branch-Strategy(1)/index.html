Git Branch Strategy (1)

2019-04-06
Git

브랜치 전략을 잘 세우면

  • 코드의 변경 이력을 효율적으로 관리 할 수 있다.
  • 프로젝트 별 히스토리를 한 눈에 파악하기 쉽다.
  • 적절한 브랜치 사용은 배포의 편의성을 제공한다.

상용 브랜치 모델

  • git-flow
  • github-flow
  • gitlab-flow

해당 모델을 적용할 때는 각자 모델을 참고하되 현재 정책, 시스템에 맞게 커스터마이징하여 사용하는 것이 바람직!

배포 환경

  • 현직장의 경우, 일주일에 최대 4회로 배포가 잦은 편
  • 프로젝트 별 개발 일정이 겹치는 경우도 간혹 있음
  • 대부분 개발 시점과 배포 시점이 동일하지 않음
    위의 이유로 deploy 라는 별도의 브랜치를 두어 반영하고자 하는 커밋 내역만 cherry-pick 하여 배포하고 있음

개발 환경

  • 작업 순서 : 디자인 > 퍼블리싱 > 개발 > QA > QA 수정 > 배포
  • SourceTree 사용
  • 배포/개발 환경을 고려하여 git-flow 모델을 적절히 활용하기로 함
