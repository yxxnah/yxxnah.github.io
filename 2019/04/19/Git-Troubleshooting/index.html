Git Troubleshooting

2019-04-19
Git

git을 사용하면서 자주 부딪혔던 상황들에 대해 정리해보았다.

git 사용 시 자주 만나는 상황들

  1. git push 가 안됨
  • 원인 : remote 저장소와 로컬 저장소가 서로 다른 기록을 가지고 있어서

  • 해결 방법
    (1) [커밋 전] remote와 동기화 후 코드 수정

    (BEST) pull upstream > push origin > 작업 > commit > push
    (2) [커밋 후] 이미 커밋한 경우

    • 리모트의 기록을 다운받아옴

    • git rebase origin/HEAD (리모트의 가장 최신 기록에 변경사항을 덮음)

    • 이 두가지가 모두 안되는 경우, 서로 다른 사람이 코드의 같은 부분을 다르게 고친 것 (둘 중 어느 변경사항을 적용할지 선택하여 커밋을 수정해야만 함)
  1. commit 취소해야 할 때

  • 해결 방법 : git reset 사용

  • 옵션

    • soft : 모든 로컬 변경사항을 유지
    • hard : 모든 작업 상태 내 변경 사항을 버림
    • mixed : 작업 상태는 그대로 두지만 인덱스는 리셋
  1. push까지 했는데 취소해야 할 때
  • 해결 방법 : git revert 사용

  1. commit을 이미 했는데, message 수정이 필요할 때

  2. push까지 했는데, message 수정이 필요할 때
